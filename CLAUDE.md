Stage 1: create the placeholder website
I am a beginner and this is my first vibe-coded website.

I run a UK handmade macramé business called Knot Perfect. I already have a domain name, a Facebook business page and an Etsy shop.

I want you to help me build the website gradually. Do not attempt to build an online shop yet.

For the first version, create a simple, polished placeholder website using Astro and TypeScript.

The website should:

1. Be mobile-first and responsive.
2. Have a warm, handmade, natural appearance suitable for a macramé business.
3. Use accessible semantic HTML.
4. Include:
   - a header with the Knot Perfect name and space for my logo;
   - a hero section with a short introduction;
   - space for a large photograph of one of my wall hangings;
   - a short paragraph explaining that I make handmade macramé wall hangings, gifts and custom pieces;
   - a prominent Etsy button;
   - a prominent Facebook button;
   - an email contact link;
   - a “Full website coming soon” message;
   - a simple footer;
   - a basic privacy page.
5. Avoid unnecessary libraries and complicated animations.
6. Store my Facebook, Etsy and email details in one clearly labelled configuration file so I can easily change them.
7. Use placeholder URLs and clearly tell me where to replace them.
8. Include suitable image placeholders, but do not invent product photographs.
9. Include clear instructions in a README explaining:
   - how to run the website locally;
   - where to add my logo and photographs;
   - where to change the text and links;
   - how to build the production version.
10. Use UK English throughout.

Before making changes:
- explain the proposed file structure;
- tell me which commands you intend to run;
- ask for permission before installing any unexpected package.

After building it:
- run the website;
- check for build errors;
- explain how I can preview it;
- summarise every file you created or changed.

Please create the Astro project in the current folder.

Stage 2

Please prepare this project for GitHub.

1. Check that the .gitignore file excludes node_modules, build output, environment variables and operating-system files.
2. Check that no passwords, API keys or personal secrets are present.
3. Initialise a Git repository if one does not already exist.
4. Create an initial commit with an appropriate message.
5. Give me beginner-friendly instructions for creating a new GitHub repository and pushing this project to it.

Do not make the GitHub repository public unless I explicitly choose to do so.
